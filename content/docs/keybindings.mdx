# Keybindings Reference

Complete keyboard shortcut reference for TUIOS. All keybindings are customizable through the [configuration file](/docs/configuration).

<Callout title="Quick Tip" type="tip">
Press <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>?</kbd> from anywhere in TUIOS to see the help overlay!
</Callout>

## Modes

Mode switching is configured in `[keybindings.mode_control]`. TUIOS has two main modes:

- **Window Management Mode** - Navigate and manage windows (default on startup)
- **Terminal Mode** - Input goes directly to the focused terminal

### Mode Switching

- <kbd>i</kbd> or <kbd>Enter</kbd> - Enter Terminal Mode
- <kbd>Alt</kbd>+<kbd>Esc</kbd> or <kbd>Opt</kbd>+<kbd>Esc</kbd> - Exit Terminal Mode (direct shortcut)
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>Esc</kbd> - Return to Window Management Mode (from Terminal Mode)
- <kbd>?</kbd> (Window Mode) or <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>?</kbd> (universal) - Toggle help overlay
- <kbd>q</kbd> (Window Mode) or <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>q</kbd> (universal) - Quit TUIOS

<Callout title="Esc and d are not the same thing" type="warn">
<kbd>Ctrl</kbd>+<kbd>B</kbd> <kbd>Esc</kbd> leaves terminal mode and returns you to window mode. It never detaches.

<kbd>Ctrl</kbd>+<kbd>B</kbd> <kbd>d</kbd> detaches from a daemon session, leaving it running in the background. These were one action in older versions and are now separate: `prefix_exit_mode` and `prefix_detach`.
</Callout>

### Terminal Mode Shortcuts

These shortcuts work directly in Terminal Mode without requiring the prefix key. Configured in `[keybindings.terminal_mode]`.

| Platform | Next Window | Previous Window | Exit Terminal Mode |
|----------|-------------|-----------------|-------------------|
| macOS | <kbd>Opt</kbd>+<kbd>Tab</kbd> | <kbd>Opt</kbd>+<kbd>Shift</kbd>+<kbd>Tab</kbd> | <kbd>Opt</kbd>+<kbd>Esc</kbd> |
| Linux/Windows | <kbd>Alt</kbd>+<kbd>N</kbd> | <kbd>Alt</kbd>+<kbd>P</kbd> | <kbd>Alt</kbd>+<kbd>Esc</kbd> |

<Callout title="Why different keybindings?" type="info">
On Linux and Windows, <kbd>Alt</kbd>+<kbd>Tab</kbd> is captured by the OS window switcher, so TUIOS uses <kbd>Alt</kbd>+<kbd>N</kbd>/<kbd>Alt</kbd>+<kbd>P</kbd> instead. On macOS, <kbd>Opt</kbd>+<kbd>Tab</kbd> produces a unicode character that TUIOS can intercept.
</Callout>

## Window Management

Window mode, from `[keybindings.window_management]` (and `[keybindings.restore_minimized]` for the last row).

- <kbd>z</kbd> - Toggle zoom (fullscreen focused window). Shared borders hidden when zoomed, dockbar shows **Z** indicator.
- <kbd>n</kbd> - Create new window
- <kbd>w</kbd> or <kbd>x</kbd> - Close focused window
- <kbd>r</kbd> - Rename focused window
- <kbd>m</kbd> - Minimize focused window
- <kbd>Shift</kbd>+<kbd>M</kbd> - Restore all minimized windows
- <kbd>Tab</kbd> - Focus next window
- <kbd>Shift</kbd>+<kbd>Tab</kbd> - Focus previous window
- <kbd>1</kbd>-<kbd>9</kbd> - Select window by number
- <kbd>Shift</kbd>+<kbd>1</kbd>-<kbd>9</kbd> or <kbd>!@#$%^&*(</kbd> - Restore minimized window by number

## Workspaces

TUIOS supports **9 workspaces** for organizing windows. Configured in `[keybindings.workspaces]`.

- <kbd>Alt</kbd>+<kbd>1</kbd> through <kbd>Alt</kbd>+<kbd>9</kbd> - Switch to workspace 1-9
- <kbd>Alt</kbd>+<kbd>Shift</kbd>+<kbd>1</kbd> through <kbd>Alt</kbd>+<kbd>Shift</kbd>+<kbd>9</kbd> - Move window to workspace and follow

<Callout title="macOS Users" type="info">
Use <kbd>Option</kbd>+<kbd>1</kbd> through <kbd>Option</kbd>+<kbd>9</kbd> (automatically configured by default)
</Callout>

## Window Layout

### Manual Snapping (Non-Tiling Mode)

#### Keyboard Snapping

- <kbd>h</kbd> - Snap window to left half
- <kbd>l</kbd> - Snap window to right half
- <kbd>f</kbd> - Fullscreen window
- <kbd>u</kbd> - Unsnap/restore window
- <kbd>1</kbd> - Snap to top-left corner
- <kbd>2</kbd> - Snap to top-right corner
- <kbd>3</kbd> - Snap to bottom-left corner
- <kbd>4</kbd> - Snap to bottom-right corner

#### Mouse Edge Snapping

In floating mode (non-tiling), drag a window to the screen edges to snap it:

| Edge | Action |
|------|--------|
| Top center | Fullscreen |
| Left edge | Snap to left half |
| Right edge | Snap to right half |
| Top-left corner | Snap to top-left quarter |
| Top-right corner | Snap to top-right quarter |
| Bottom-left corner | Snap to bottom-left quarter |
| Bottom-right corner | Snap to bottom-right quarter |

<Callout title="Edge Detection" type="tip">
Simply drag a window by its title bar and release when the cursor reaches the screen edge (within 5 pixels of the edge).
</Callout>

### Tiling Mode

TUIOS uses Binary Space Partitioning (BSP) for automatic tiling. Windows are arranged in an alternating vertical/horizontal split pattern (spiral layout).

- <kbd>t</kbd> - Toggle automatic tiling mode
- <kbd>Shift</kbd>+<kbd>H</kbd> or <kbd>Ctrl</kbd>+<kbd>Left</kbd> - Swap with window to the left
- <kbd>Shift</kbd>+<kbd>L</kbd> or <kbd>Ctrl</kbd>+<kbd>Right</kbd> - Swap with window to the right
- <kbd>Shift</kbd>+<kbd>K</kbd> or <kbd>Ctrl</kbd>+<kbd>Up</kbd> - Swap with window above
- <kbd>Shift</kbd>+<kbd>J</kbd> or <kbd>Ctrl</kbd>+<kbd>Down</kbd> - Swap with window below
- <kbd>&lt;</kbd> or <kbd>Shift</kbd>+<kbd>,</kbd> - Decrease master window width (from right edge)
- <kbd>&gt;</kbd> or <kbd>Shift</kbd>+<kbd>.</kbd> - Increase master window width (from right edge)
- <kbd>&#123;</kbd> or <kbd>Shift</kbd>+<kbd>[</kbd> - Decrease focused window height (from bottom edge)
- <kbd>&#125;</kbd> or <kbd>Shift</kbd>+<kbd>]</kbd> - Increase focused window height (from bottom edge)
- <kbd>.</kbd> - Increase master window width from the left edge
- <kbd>[</kbd> - Decrease focused window height from the top edge
- <kbd>]</kbd> - Increase focused window height from the top edge

<Callout title="One default binding you cannot reach" type="warn">
`resize_master_shrink_left` is bound to a bare <kbd>,</kbd> by default, and you will never trigger it. In window mode, <kbd>,</kbd> is intercepted for the settings page before any keybinding lookup runs.

Rebinding <kbd>,</kbd> in `[keybindings.layout]` does not help either, because the intercept is not registry-driven. Bind `resize_master_shrink_left` to a different key if you want it.
</Callout>

### BSP Split Controls

In window mode, from `[keybindings.layout]`:

- <kbd>-</kbd> - `split_horizontal`
- <kbd>&#124;</kbd> or <kbd>&#92;</kbd> - `split_vertical`
- <kbd>R</kbd> - `rotate_split`
- <kbd>=</kbd> - `equalize_splits`
- <kbd>Alt</kbd>+<kbd>h</kbd> <kbd>Alt</kbd>+<kbd>j</kbd> <kbd>Alt</kbd>+<kbd>k</kbd> <kbd>Alt</kbd>+<kbd>l</kbd> - `preselect_left` / `preselect_down` / `preselect_up` / `preselect_right`, choosing where the next window lands (<kbd>Opt</kbd> on macOS)

The same four split operations are also on the prefix key, as separate actions in `[keybindings.prefix_mode]`:

- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>-</kbd> - `prefix_split_horizontal`
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>&#124;</kbd> or <kbd>&#92;</kbd> - `prefix_split_vertical`
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>R</kbd> - `prefix_rotate_split`
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>=</kbd> - `prefix_equalize_splits`

Rebinding one does not rebind the other. They are two independent sets.

<Callout title="Split Indicator" type="tip">
The dock shows the next split direction (V for vertical, H for horizontal) when tiling mode is active.
</Callout>

## Copy Mode

Enter copy mode with <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>[</kbd> to navigate scrollback and select text using vim-style commands.

<Callout title="Vim Users" type="tip">
Copy mode supports 50+ vim motions including word movements, paragraph jumps, and search!
</Callout>

### Basic Navigation

- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>[</kbd> - Enter copy mode
- <kbd>h</kbd> <kbd>j</kbd> <kbd>k</kbd> <kbd>l</kbd> - Move cursor left/down/up/right
- <kbd>w</kbd> <kbd>b</kbd> <kbd>e</kbd> - Word forward / word backward / word end
- <kbd>0</kbd> <kbd>^</kbd> <kbd>$</kbd> - Start of line / first non-blank / end of line
- <kbd>g</kbd><kbd>g</kbd> - Jump to top of scrollback
- <kbd>G</kbd> - Jump to bottom (live output)
- <kbd>&#123;number&#125;</kbd><kbd>G</kbd> - Jump to line number (e.g., 10G)
- <kbd>&#123;</kbd> <kbd>&#125;</kbd> - Jump to previous/next paragraph
- <kbd>Ctrl</kbd>+<kbd>U</kbd> <kbd>Ctrl</kbd>+<kbd>D</kbd> - Half page up/down
- <kbd>Ctrl</kbd>+<kbd>B</kbd> <kbd>Ctrl</kbd>+<kbd>F</kbd> - Full page up/down
- <kbd>i</kbd> - Return to terminal mode
- <kbd>q</kbd> or <kbd>Esc</kbd> - Exit copy mode

### Count Prefix

Prefix any motion with a number to repeat it:
- <kbd>1</kbd><kbd>0</kbd><kbd>j</kbd> - Move down 10 lines
- <kbd>5</kbd><kbd>w</kbd> - Move forward 5 words
- <kbd>3</kbd><kbd>&#123;</kbd> - Jump up 3 paragraphs

### Character Search

- <kbd>f</kbd><kbd>&#123;char&#125;</kbd> - Find next occurrence of char on line
- <kbd>F</kbd><kbd>&#123;char&#125;</kbd> - Find previous occurrence of char on line
- <kbd>t</kbd><kbd>&#123;char&#125;</kbd> - Move cursor before next char
- <kbd>T</kbd><kbd>&#123;char&#125;</kbd> - Move cursor after previous char
- <kbd>;</kbd> - Repeat last character search
- <kbd>,</kbd> - Repeat last search (opposite direction)

### Search

- <kbd>/</kbd> - Search forward
- <kbd>?</kbd> - Search backward
- <kbd>n</kbd> - Next match
- <kbd>N</kbd> - Previous match
- <kbd>Ctrl</kbd>+<kbd>L</kbd> - Clear search highlights

### Visual Selection

- <kbd>v</kbd> - Enter visual character mode
- <kbd>V</kbd> - Enter visual line mode
- <kbd>y</kbd> or <kbd>c</kbd> - Yank (copy) selection to clipboard
- <kbd>Esc</kbd> or <kbd>q</kbd> - Exit visual mode

## Command Palette

- <kbd>Ctrl</kbd>+<kbd>P</kbd> - Open Command Palette (works from any mode)
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>P</kbd> - Open Command Palette (prefix alternative)

The command palette provides fuzzy-searchable access to all TUIOS commands, keybindings, and actions.

## Prefix Commands

Press <kbd>Ctrl</kbd>+<kbd>B</kbd>, release, then press the command key (tmux-style).

<Callout title="Leader Key" type="info">
The leader key (<kbd>Ctrl</kbd>+<kbd>B</kbd> by default) is configurable. See [Configuration Guide](/docs/configuration) for details.
</Callout>

### Main Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd>)

Every row below is an action in `[keybindings.prefix_mode]` and can be rebound.

| Key | Action name | Does |
|-----|-------------|------|
| <kbd>c</kbd> | `prefix_new_window` | Create new window |
| <kbd>x</kbd> | `prefix_close_window` | Close current window |
| <kbd>r</kbd> | `prefix_rename_window` | Rename window |
| <kbd>,</kbd> | `prefix_settings` | Open the settings page |
| <kbd>n</kbd> or <kbd>Tab</kbd> | `prefix_next_window` | Next window |
| <kbd>p</kbd> or <kbd>Shift</kbd>+<kbd>Tab</kbd> | `prefix_prev_window` | Previous window |
| <kbd>0</kbd>-<kbd>9</kbd> | `prefix_select_0` to `prefix_select_9` | Jump to window |
| <kbd>Space</kbd> | `prefix_toggle_tiling` | Toggle tiling mode |
| <kbd>z</kbd> | `prefix_fullscreen` | Toggle zoom on current pane |
| <kbd>-</kbd> | `prefix_split_horizontal` | Split horizontally (top/bottom) |
| <kbd>&#124;</kbd> or <kbd>&#92;</kbd> | `prefix_split_vertical` | Split vertically (left/right) |
| <kbd>R</kbd> | `prefix_rotate_split` | Rotate split direction |
| <kbd>=</kbd> | `prefix_equalize_splits` | Equalize all split ratios |
| <kbd>[</kbd> | `prefix_selection` | Enter copy mode |
| <kbd>s</kbd> | `prefix_scrollback` | Open the scrollback browser |
| <kbd>S</kbd> | `prefix_session_switcher` | Open the session switcher |
| <kbd>P</kbd> | `prefix_command_palette` | Open the command palette |
| <kbd>L</kbd> | `prefix_layout` | Enter layout prefix menu |
| <kbd>w</kbd> | `prefix_workspace` | Enter workspace prefix menu |
| <kbd>m</kbd> | `prefix_minimize` | Enter minimize prefix menu |
| <kbd>t</kbd> | `prefix_window` | Enter window prefix menu |
| <kbd>D</kbd> | `prefix_debug` | Enter debug prefix menu |
| <kbd>T</kbd> | `prefix_tape` | Enter tape prefix menu |
| <kbd>d</kbd> | `prefix_detach` | Detach; a daemon session keeps running |
| <kbd>Esc</kbd> | `prefix_exit_mode` | Leave terminal mode, never detaches |
| <kbd>q</kbd> | `prefix_quit` | Quit; in daemon mode this kills the session |
| <kbd>?</kbd> | `prefix_help` | Toggle help |

<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>Ctrl</kbd>+<kbd>B</kbd> sends a literal <kbd>Ctrl</kbd>+<kbd>B</kbd> to the terminal.

<Callout title="Copy mode and the scrollback browser are different" type="info">
<kbd>Ctrl</kbd>+<kbd>B</kbd> <kbd>[</kbd> enters copy mode, with vim motions and text selection. <kbd>Ctrl</kbd>+<kbd>B</kbd> <kbd>s</kbd> opens the scrollback browser, which is for reading back through history. They are separate actions with separate bindings.
</Callout>

### Window Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>t</kbd>)

Configured in `[keybindings.window_prefix]`.

- <kbd>n</kbd> - `window_prefix_new`, create new window
- <kbd>x</kbd> - `window_prefix_close`, close current window
- <kbd>r</kbd> - `window_prefix_rename`, rename window
- <kbd>Tab</kbd> - `window_prefix_next`, next window
- <kbd>Shift</kbd>+<kbd>Tab</kbd> - `window_prefix_prev`, previous window
- <kbd>t</kbd> - `window_prefix_tiling`, toggle tiling mode
- <kbd>Esc</kbd> - `window_prefix_cancel`, cancel

### Minimize Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>m</kbd>)

Configured in `[keybindings.minimize_prefix]`.

- <kbd>m</kbd> - `minimize_prefix_focused`, minimize the focused window
- <kbd>1</kbd>-<kbd>9</kbd> - `minimize_prefix_restore_1` to `minimize_prefix_restore_9`, restore minimized window by position
- <kbd>M</kbd> - `minimize_prefix_restore_all`, restore all minimized windows
- <kbd>Esc</kbd> - `minimize_prefix_cancel`, cancel

### Tape Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>T</kbd>)

Record and replay `.tape` automation scripts. Configured in `[keybindings.tape_prefix]`.

- <kbd>m</kbd> - `tape_prefix_manager`, open the tape manager
- <kbd>r</kbd> - `tape_prefix_record`, start recording
- <kbd>s</kbd> - `tape_prefix_stop`, stop recording
- <kbd>Esc</kbd> - `tape_prefix_cancel`, cancel

### Workspace Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>w</kbd>)

Configured in `[keybindings.workspace_prefix]`.

- <kbd>1</kbd>-<kbd>9</kbd> - `workspace_prefix_switch_1` to `workspace_prefix_switch_9`, switch to workspace
- <kbd>!</kbd> <kbd>@</kbd> <kbd>#</kbd> <kbd>$</kbd> <kbd>%</kbd> <kbd>^</kbd> <kbd>&</kbd> <kbd>*</kbd> <kbd>(</kbd> - `workspace_prefix_move_1` to `workspace_prefix_move_9`, move window to workspace and follow
- <kbd>Esc</kbd> - `workspace_prefix_cancel`, cancel

### Debug Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>D</kbd>)

Debug and development tools. Configured in `[keybindings.debug_prefix]`.

- <kbd>l</kbd> - `debug_prefix_logs`, toggle log viewer
- <kbd>c</kbd> - `debug_prefix_cache`, toggle cache statistics
- <kbd>k</kbd> - `debug_prefix_showkeys`, toggle showkeys overlay
- <kbd>a</kbd> - `debug_prefix_animations`, toggle animations
- <kbd>Esc</kbd> - `debug_prefix_cancel`, cancel

### Layout Prefix (<kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>L</kbd>)

Save and load window layout templates:

- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>L</kbd> then <kbd>l</kbd> - Load layout template
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>L</kbd> then <kbd>s</kbd> - Save layout template
- <kbd>Ctrl</kbd>+<kbd>B</kbd> then <kbd>L</kbd> then <kbd>Esc</kbd> - Cancel

Layout loading is non-destructive: existing windows are repositioned to match the template rather than being killed. Extra windows are minimized.

<Callout title="The only prefix menu you cannot rebind" type="info">
Entering this menu is the configurable `prefix_layout` action, but <kbd>l</kbd> and <kbd>s</kbd> inside it are literal. The layout sub-prefix has no config section of its own.
</Callout>

## Mouse Controls

- **Left Click** - Focus window
- **Left Drag** - Move window (non-tiling) or swap windows (tiling)
- **Right Drag** - Resize window (non-tiling only)
- **Title Bar Buttons** - Minimize, maximize, or close window
- **Click Dock Item** - Restore minimized window
- **Copy Mode Click** - Move cursor to position
- **Copy Mode Drag** - Select text (enters visual mode). Dragging above/below the pane auto-scrolls via a timer.
- **Mouse Wheel Up** - Enter copy mode and scroll up (when the application is not tracking the mouse and is not on the alternate screen)
- **Right Border Click/Drag** - Interactive scrollbar for navigating scrollback history

## Customization

Every binding on this page lives in one of 14 sections under `[keybindings]` in `config.toml`, and every one is rebindable. The section is named next to each group above. See the [Configuration Guide](/docs/configuration) for the full list and the key syntax.

```bash
# Edit your keybindings
tuios config edit

# View current configuration
tuios keybinds list

# View your customizations
tuios keybinds list-custom
```

<Callout title="Prefix bindings work now" type="info">
Six sections used to be parsed and then ignored, so rebinding anything under the leader key silently did nothing: `prefix_mode`, `window_prefix`, `minimize_prefix`, `workspace_prefix`, `debug_prefix`, and `tape_prefix`. They all resolve properly now, as does `terminal_mode`. If you gave up on customizing your prefix menus, try again.
</Callout>

<Callout title="list-custom is not a full audit" type="warn">
`tuios keybinds list-custom` compares 9 of the 14 sections. It ignores `navigation`, `restore_minimized`, `debug_prefix`, `tape_prefix`, and `terminal_mode`, and it never reports a customized `leader_key`. A config that changes only those prints "No custom keybindings configured".
</Callout>

<Callout title="Most keybinding changes need a restart" type="warn">
TUIOS hot-reloads part of `[appearance]`, but keybindings are not in that subset. Restart the client after editing them.
</Callout>

## Platform-Specific Notes

### macOS

Default workspace switching uses Option key:
- <kbd>Option</kbd>+<kbd>1</kbd> through <kbd>Option</kbd>+<kbd>9</kbd> - Switch workspace
- <kbd>Option</kbd>+<kbd>Shift</kbd>+<kbd>1</kbd> through <kbd>Option</kbd>+<kbd>Shift</kbd>+<kbd>9</kbd> - Move window to workspace

Terminal mode shortcuts:
- <kbd>Opt</kbd>+<kbd>Tab</kbd> / <kbd>Opt</kbd>+<kbd>Shift</kbd>+<kbd>Tab</kbd> - Cycle windows
- <kbd>Opt</kbd>+<kbd>Esc</kbd> - Exit terminal mode

You can still type Option key unicode characters (¡™£¢∞§¶•ª) in Terminal Mode.

### Linux/Windows

Uses standard Alt key for workspace switching:
- <kbd>Alt</kbd>+<kbd>1</kbd> through <kbd>Alt</kbd>+<kbd>9</kbd>
- <kbd>Alt</kbd>+<kbd>Shift</kbd>+<kbd>1</kbd> through <kbd>Alt</kbd>+<kbd>Shift</kbd>+<kbd>9</kbd>

Terminal mode shortcuts:
- <kbd>Alt</kbd>+<kbd>N</kbd> / <kbd>Alt</kbd>+<kbd>P</kbd> - Cycle windows (next/previous)
- <kbd>Alt</kbd>+<kbd>Esc</kbd> - Exit terminal mode

<Callout title="Note" type="info">
<kbd>Alt</kbd>+<kbd>Tab</kbd> is not used on Linux/Windows because it conflicts with the OS window switcher.
</Callout>
