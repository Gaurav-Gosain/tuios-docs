# Getting Started with TUIOS

This guide will help you install TUIOS and learn the essential operations.

<Callout title="Requirements" type="info">
- A terminal with true color support (most modern terminals)
- Go 1.24+ (if building from source)
</Callout>

## First Launch

Simply run:

```bash
tuios
```

<Callout title="No windows yet?" type="info">
You'll start in **Window Management Mode** with no windows. This is normal - TUIOS lets you create windows as needed.
</Callout>

## Understanding Modes

TUIOS has **two primary modes**:

### 1. Window Management Mode 

The mode you start in. This is where you:
- Navigate between windows
- Create/close/rename windows  
- Switch workspaces
- Tile windows
- Use mouse to drag/resize

**Visual Indicator**: Windows have colored borders and you see the dockbar at bottom.

### 2. Terminal Mode

Where you interact with the terminal inside a window. Your keypresses go directly to the shell/application.

**Visual Indicator**: The active window is fully focused, keyboard input is captured.

<Callout title="Mode Switching" type="tip">
- Press <kbd>i</kbd> or <kbd>Enter</kbd> to enter **Terminal Mode**
- Press <kbd>Esc</kbd> or <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>d</kbd> to return to **Window Management Mode**
</Callout>

## Essential Keybindings

### Creating Your First Window

From Window Management Mode:
- Press <kbd>n</kbd> to create a new window

Or from any mode:
- Press <kbd>Ctrl</kbd><kbd>B</kbd>, then <kbd>c</kbd>

### Window Navigation

In Window Management Mode:
- <kbd>h</kbd><kbd>j</kbd><kbd>k</kbd><kbd>l</kbd> - Move focus (vim-style)
- <kbd>Tab</kbd> - Next window
- <kbd>Shift</kbd><kbd>Tab</kbd> - Previous window

Or click with your mouse!

### Window Management

From any mode (using <kbd>Ctrl</kbd><kbd>B</kbd> prefix):
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>c</kbd> - Create new window
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>x</kbd> - Close focused window
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>r</kbd> - Rename focused window
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>m</kbd> - Minimize focused window

### Workspaces

TUIOS provides **9 workspaces** (like virtual desktops):

- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>1</kbd>-<kbd>9</kbd> - Switch to workspace 1-9
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>Shift</kbd><kbd>1</kbd>-<kbd>9</kbd> - Move window to workspace and follow

### Tiling

- <kbd>t</kbd> - Toggle automatic tiling mode
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>f</kbd> - Fullscreen focused window
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>h</kbd> - Snap window left (50% width)
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>l</kbd> - Snap window right (50% width)

### Copy Mode (Scrollback)

Access 10,000 lines of scrollback with vim-style navigation:

- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>[</kbd> - Enter copy mode
- <kbd>h</kbd><kbd>j</kbd><kbd>k</kbd><kbd>l</kbd> - Move cursor
- <kbd>/</kbd> - Search forward
- <kbd>v</kbd> - Start visual selection
- <kbd>y</kbd> - Copy selection to clipboard
- <kbd>Esc</kbd> - Exit copy mode

<Callout title="More keybindings" type="info">
See the [Keybindings](/docs/keybindings) page for the complete reference.
</Callout>

### Help & Quit

- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>?</kbd> - Toggle help overlay
- <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>q</kbd> - Quit TUIOS

## Example Workflow

Let's create a development setup:

<Steps>

<Step>

### Launch TUIOS

```bash
tuios
```

You start in Window Management Mode with no windows.

</Step>

<Step>

### Create first window

Press <kbd>n</kbd>

A new window appears with your shell.

</Step>

<Step>

### Enter terminal mode

Press <kbd>i</kbd> or <kbd>Enter</kbd>

Now you can type commands.

</Step>

<Step>

### Run a command

```bash
ls
vim myfile.go
```

Work normally in the terminal.

</Step>

<Step>

### Return to management mode

Press <kbd>Esc</kbd>

Back to Window Management Mode.

</Step>

<Step>

### Create more windows

Press <kbd>n</kbd> twice more.

You now have 3 windows!

</Step>

<Step>

### Enable tiling

Press <kbd>t</kbd>

Windows automatically arrange in a grid.

</Step>

<Step>

### Switch workspaces

Press <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>2</kbd>

Move to empty workspace 2.

</Step>

<Step>

### Create windows here too

Press <kbd>n</kbd> a few times.

Each workspace is independent!

</Step>

<Step>

### Navigate between windows

- Press <kbd>Tab</kbd> to cycle windows
- Press <kbd>Ctrl</kbd><kbd>B</kbd> <kbd>1</kbd> to go back to workspace 1
- Click windows with your mouse

</Step>

</Steps>

## Command Line Options

TUIOS supports many flags:

```bash
# Run with a specific theme
tuios --theme dracula

# List all available themes (300+)
tuios --list-themes

# Preview a theme's colors
tuios --preview-theme nord

# Enable debug mode
tuios --debug

# Use ASCII-only (no Nerd Font icons)
tuios --ascii-only

# Show pressed keys overlay (for demos)
tuios --show-keys

# Custom border style
tuios --border-style thick

# Custom dockbar position
tuios --dockbar-position top
```

## SSH Server Mode

Run TUIOS as an SSH server for remote access:

```bash
# Start on default port (2222)
tuios ssh

# Custom port
tuios ssh --port 8022

# Listen on all interfaces
tuios ssh --host 0.0.0.0 --port 2222
```

Connect from another machine:

```bash
ssh -p 2222 user@hostname
```

Each SSH connection gets its own isolated TUIOS session!

## Configuration

TUIOS stores its configuration in a TOML file:

```bash
# Edit configuration
tuios config edit

# Print config file path
tuios config path

# Reset to defaults
tuios config reset

# List all keybindings
tuios keybinds list

# List only custom keybindings
tuios keybinds list-custom
```

See the [Configuration](/docs/configuration) guide for details.

## Shell Completions

Generate shell completions for your shell:

```bash
# Bash
tuios completion bash > /etc/bash_completion.d/tuios

# Zsh
tuios completion zsh > "${fpath[1]}/_tuios"

# Fish
tuios completion fish > ~/.config/fish/completions/tuios.fish

# PowerShell
tuios completion powershell > tuios.ps1
```
